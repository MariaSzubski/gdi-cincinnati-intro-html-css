SECTION 1

Box Sizing

Box-Sizing

Box-sizing tells the browser how to calculate the
width and height of an object.

By default, the height and width of an object includes only the objects content area.

Box-Sizing: Content-Box

content-box is the default setting for box-sizing

Width & height measurements include only the content box.

html {
  box-sizing: content-box;
}

Box-Sizing: Border-Box

The alternative is border-box

Width & height measurements include the content box, padding, and border in total.

html {
  box-sizing: border-box;
}

Box-Sizing

While content-box is the default, border-box can sometimes be easier to conceptualize and set values for, and you might want to use it instead when styling your sites.

Box-Sizing

For example, when trying to craft a four equal columns with 24px of padding:

.column {
  box-sizing: content-box;
  width: 19%;
  padding: 3%;
  /* Total column width = 25% */
}
.column {
  box-sizing: border-box;
  width: 25%;
  padding: 3%;
  /* Total column width = 25% */
}

Relative Positioning

{ position: relative; }
  • Allows an element to be moved away from it's normal position to the top, left, right or bottom.
  • Does not affect the position of the elements surrounding it.
  • Turns an element into a "positioning context", against which other elements can be positioned relative to it.
  • Relative positioning and absolute positioning are used together.

Relative Positioning

The relative value will leave the element in the normal flow, but then offset it according to the top, left, right, or bottom properties.

<div class="static">
  <div class="relative">
    Hello, hi!
  </div>
</div>








.static {
  position: static;
  border: 2px solid orange;
}

.relative{
    position: relative;
    left: 80px;
    top: 20px;
    height: 100px;
    background-color: aquamarine;
  }

Absolute Positioning

html { position: absolute; }
  • Positions an element outside of the normal flow.
  • It is offset from the nearest positioned ancestor.
    (Any container block with a position other than static)
  • If no such element is found, the container block is <html>.
  • Other elements act as if it's not there.

Absolute Positioning

The absolute value will take the element out of the normal flow and position it in relation to the nearest positioned container.

<div class="container">
    <div class="top">
        Up here
    </div>
    <div class="bottom">
        Down here
    </div>
</div>








.container {
  position: relative;
  border: 1px solid orange;

  right: 10px;

Z-index

When you use positioning to move elements out of the normal flow of content, elements can overlap.

You can change the order of overlapping with z-index. The element with highest z-index goes on top.

<div class="container">
  <div class="first">
    First
  </div>
  <div class="second">
    Second
  </div>
</div>
.second {
  position: absolute;
  bottom: 10px;
  left: 80px;
  background-color: yellow;
}

.first {
  position: absolute;
  bottom: 25px;
  left:60px;
  background-color: green;
  z-index: 2;
}

Fixed Positioning

Fixed positioning is sort of like absolute positioning... but it positions your element relative to the browser window. 

img{
  position: fixed;
  bottom: 80px;
  right: 0px;
  /* This will 'fix' the image to the bottom of the browser window
  and allow elements to scroll beneath it. */
}

Float

  • "Floating" an element pulls it to the far left or right of it's containing element.
  • Any other elements, such as paragraphs or lists, will wrap around the floated element.
  • Always specify a width when floating an element, otherwise the element is likely to take up the whole page and not appear floated.
  • You can specify a) whether an element is floated or not, and b) which side it ï¬‚oats on.
